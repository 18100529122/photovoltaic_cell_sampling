# 问题发现记录

## 1. FreeRTOS堆配置
- configTOTAL_HEAP_SIZE = 32768 字节 (32KB)
- defaultTask 栈大小: 4096 * 4 = 16384 字节 (16KB)
- adc_process 任务栈: 2048 字 = 8192 字节
- 已用堆: 16384 + 8192 = 24576 字节 (约75%)
- 当 adc_process 栈设为 4096 字时 = 16384 字节，总需求 16384+16384=32768 字节，刚好用完，没有剩余空间供内部使用 → 创建失败

## 2. ADC数据结构内存占用
- ADC_Data_t 结构非常大:
  - vs[12]: 12 * (200*2 + 2 + 1) = 4836 字节
  - cur[12]: 4836 字节
  - temp[2]: 2 * (400 + 2 + 1) = 806 字节
  - 结果 float: (12+12+2)*4 = 104 字节
  - 总计: ~10582 字节 (全局变量，不在堆上)

## 3. TIM2卡住的可能原因
- 问题发生在 HAL_TIM_Base_Start(&htim2)
- 可能的原因:
  1. 中断优先级配置问题
  2. printf 缓冲区溢出/死锁
  3. 栈溢出 (2048 字可能不够)
  4. 任务优先级问题

## 4. HAL_ADC_Start_DMA 分析
- 这4个 ADC DMA 启动成功，说明 HAL 库在这些调用时没有问题
- 问题在 TIM2 启动处
