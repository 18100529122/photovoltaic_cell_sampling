# 光伏电池采样系统 - 项目计划

## 项目概述

基于 STM32G474VETx + FreeRTOS 的光伏电池多通道采样系统，实现26路模拟信号采集、数据处理、CAN通信及参数存储功能。

---

## 功能需求

### 1. ADC采样功能
- **通道数量**: 26路模拟输入
- **采样频率**: 500kHz
- **采样策略**: 1秒内对所有通道采集200个点（可扩展）
- **数据处理**: 每通道200个点中，去掉10个最大值、10个最小值，计算平均值

### 2. CAN通信
- 发送26路电压、电流、温度值
- 通信协议待制定

### 3. 串口通信
- **UART4**: 调试串口
- **UART5**: 心跳包发送

### 4. 参数存储
- 使用片内Flash保存配置参数

---

## 硬件资源分配

| 功能 | 外设 | 说明 |
|------|------|------|
| 操作系统 | FreeRTOS | CMSIS-RTOS v2 |
| ADC采样 | ADC1-4 | 26通道，12位分辨率 |
| CAN通信 | FDCAN2 | 经典CAN格式 |
| 调试串口 | UART4 | 115200 8N1 |
| 心跳串口 | UART5 | 115200 8N1 |
| 参数存储 | 片内Flash | 需规划分区 |
| 状态指示 | LED1 (PD14) | 系统状态 |

---

## FreeRTOS 任务设计

| 任务名称 | 优先级 | 堆栈大小 | 功能描述 |
|---------|--------|---------|---------|
| ADC采集任务 | High | 512 | 管理ADC采样，处理DMA完成中断 |
| 数据处理任务 | High | 1024 | 数据滤波、去极值、平均值计算 |
| CAN发送任务 | Medium | 512 | 周期发送26路数据 |
| 心跳任务 | Low | 256 | UART5心跳包定时发送 |
| 调试任务 | Low | 512 | UART4命令处理与日志输出 |
| LED任务 | Low | 128 | LED状态指示 |

### 任务间通信机制
- **队列**: 数据采集 → 数据处理
- **队列**: 数据处理 → CAN发送
- **互斥锁**: 共享资源保护
- **事件组**: 同步多任务

## 数据流程设计

```
模拟信号 → ADC1-4 → DMA → 缓冲区(26×200) 
→ 数据处理(去极值+平均) → CAN发送
                           ↓
                        UART5心跳
```

## 关键技术点

1. **ADC多通道同步采样**: 合理分配4个ADC的通道，利用DMA提高效率
2. **数据处理实时性**: 500kHz采样率下的算法优化
3. **CAN总线带宽**: 26路数据传输的帧结构设计
4. **Flash磨损平衡**: 参数存储的可靠性设计
